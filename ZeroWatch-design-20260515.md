## Problem Statement

Critical infrastructure operators (utilities, railways, nuclear plants) are hit by
coordinated cyberattacks — the same attack pattern targeting multiple operators
simultaneously. They cannot warn each other. Sharing means disclosing network
topology, vulnerability details, and breach history to competitors, regulators who
trigger mandatory reporting, and potentially the attackers themselves.

ISACs (Information Sharing and Analysis Centers — the existing industry bodies for
threat intel sharing) are broken: federal funding was cut in 2025, trust barriers
prevent real disclosure, and sharing traffic signatures reveals infrastructure layout.
The result: operators sit on actionable threat intelligence, alone.

Zero-knowledge proofs solve this exactly. You can prove "I detected attack pattern X"
without revealing X, your network, or that you were breached. Two operators can verify
they saw the same attack — without disclosing it to each other.

**ZeroWatch is the sharing layer, not the sensor.** Detection is a commodity
(Claroty, Dragos, Nozomi own that market at $22B and growing). What doesn't exist is
a privacy-preserving channel for operators to correlate attacks across organizational
boundaries.

---

## Why This Fits Midnight

Midnight's programmable privacy (ZK proofs via Compact) is the only blockchain
primitive that enables this. You need:
- Private off-chain computation (the hash and witness)
- Public on-chain verification (the proof posted to the contract)
- Match logic that reveals nothing about the matched data

Any other approach either exposes the data or requires trusting a central party.
Midnight makes the proof the product.

## Demand Evidence

- **ISAC failure is documented**: MS-ISAC lost half its operating budget in March 2025.
  CIPAC (Critical Infrastructure Partnership Advisory Council) eliminated entirely.
- **Privacy barrier is the core problem**: Research explicitly identifies "inappropriate
  sharing could threaten privacy of companies" as why SCADA threat intel isn't shared.
- **Real coordinated attacks with no cross-operator response**: Ukraine grid (2015, 2016),
  Colonial Pipeline (2021), Triton/TRISIS (2017) — all had patterns other operators
  could have defended against, had anyone been able to share safely.
- **Gap in incumbents**: Claroty, Dragos, Nozomi do detection. None do privacy-preserving
  cross-operator correlation. Structurally impossible for them — customers won't consent
  to sharing data on a vendor's central platform.

**Honest caveat**: No direct operator interviews. First validation step after the
hackathon: find one OT security engineer, show them the demo, ask "have you been

## Premises

1. **Detection is solved — you are building the sharing layer, not the sensor.**
   Your input is a hash of an alert that any existing NIDS (Zeek, Suricata, Dragos)
   would have already generated.

2. **The ZK proof is the product.** Without the privacy guarantee, this is just
   another ISAC. The proof that "I detected X without revealing X" is what's novel.

3. **For the demo, two simulated operators on one machine is sufficient.**
   The cryptographic proof is real even if the traffic is simulated.

4. **Compact's witness mechanism is the right primitive.**
   Witnesses run off-chain with private data, generate a ZK proof, post only the
   proof on-chain.

## Architecture

```
│  Operator A                    Operator B                    │
│  ──────────                    ──────────                    │
│  SCADA sim → mock alert        SCADA sim → mock alert        │
│  {protocol, anomaly, tbucket}  {protocol, anomaly, tbucket}  │

│  witness: hash(alert) = H_A    witness: hash(alert) = H_B    │
│  ZK proof: "I know S where     ZK proof: "I know S where     │
│             hash(S) = H_A"                  hash(S) = H_B"   │
│       │                               │                      │
│       └──────────┬────────────────────┘                      │
│                  ▼                                            │
│         Midnight Contract (Compact)                           │
│         public ledger: Map of (hash → AlertRecord)           │
│         if H_A == H_B → emit MATCH event                     │
│         neither operator can see the other's raw alert        │
│                  │                                            │
│                  ▼                                            │
│    Both operators receive notification:                       │
│    "COORDINATED ATTACK DETECTED —                             │
│     1 other operator confirmed the same signature"            │
└─────────────────────────────────────────────────────────────┘
```

## Compact Contract (pseudocode)

```typescript
// Public on-chain state
ledger alertRegistry: Map<Bytes32, AlertRecord>;

struct AlertRecord {
  operatorId: Bytes32,  // hash of operator identity, not the name
  timestamp:  Uint64,
  matched:    Boolean
}

// Witness — runs OFF-CHAIN, accesses private alert data
// Raw alert never touches the blockchain
witness computeAlertHash(
  protocol:     Bytes,   // e.g. "MODBUS"
  anomalyClass: Bytes,   // e.g. "FUNCTION_CODE_SCAN"
  timeBucket:   Uint64   // Unix timestamp rounded to 15-min window
): Bytes32 {
  return hash(protocol || anomalyClass || timeBucket);
}

// Circuit — generates ZK proof that operator knows the raw alert
export circuit submitAlert(operatorId: Bytes32): [] {
  const alertHash = computeAlertHash(...);

  if (alertRegistry.contains(alertHash)) {
    alertRegistry[alertHash].matched = true;
    emit MatchDetected { alertHash, timestamp: now() };
  } else {
    alertRegistry.insert(alertHash, AlertRecord {
      operatorId,
      timestamp: now(),
      matched:   false
    });
  }
}
```

## Alert Signature Schema

Hash input: `hash(protocol_type || anomaly_class || time_bucket)`

| Field | Example values |
|-------|---------------|
| `protocol_type` | `MODBUS`, `DNP3`, `IEC61850`, `S7COMM` |
| `anomaly_class` | `FUNCTION_CODE_SCAN`, `WRITE_TO_COIL`, `UNEXPECTED_MASTER`, `READ_ALL_REGISTERS` |
| `time_bucket` | Unix timestamp rounded to nearest 900s (15-min window) |

Deliberately coarse — correlation, not forensics. Two operators who see the same
attack class within the same 15-minute window match, even if not simultaneous.

## Demo Script (3 minutes)

**Minute 1 — The problem**
"When a nuclear plant detects an intrusion, they can't tell anyone. Sharing reveals
their network to competitors, triggers regulatory liability, and tips off attackers
to what was caught. ISACs exist for this but they're effectively broken — federal
funding was cut in 2025, and the fundamental privacy problem was never solved."

**Minute 2 — The solution (live demo)**
"ZeroWatch uses Midnight's zero-knowledge proofs. Each operator's sensor generates
an alert. The alert gets hashed locally — it never leaves their machine. Only the
ZK proof goes on-chain. When two operators post the same hash, the Midnight contract
fires a match event. Neither operator sees what the other detected. They only know:
someone else saw the same thing."

[Run two-operator scenario. Both terminals submit. MATCH fires. Both dashboards light up.]

**Minute 3 — Why Midnight specifically**
"This is impossible without ZK proofs. Any other sharing mechanism either exposes
the raw data or requires trusting a central party — which operators won't do.
Midnight lets the proof be the product. The infrastructure stays private."
